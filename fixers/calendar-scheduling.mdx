## Calendar Overview

The calendar system allows fixers to manage their availability, schedule appointments with customers, and control when they can accept jobs.

<Note>
The calendar supports both desktop and mobile views with different interaction patterns optimized for each platform.
</Note>

## Accessing the Calendar

Navigate to the calendar page at `/calendar`:

```typescript
// Location: src/app/[locale]/calendar/page.tsx:19-45
export default function CalendarPage() {
  const [fixer_id, setFixerId] = useState<string>('');
  const [requester_id, setRequesterId] = useState<string>('');
  const [userRole, setUserRole] = useState<'requester' | 'fixer'>('requester');
  
  // Load data from sessionStorage
  useEffect(() => {
    const storedFixerId = sessionStorage.getItem('fixer_id');
    const storedRequesterId = sessionStorage.getItem('requester_id');
    const storedRoleUser = sessionStorage.getItem('roluser');
    
    if (storedFixerId) {
      setFixerId(storedFixerId);
      setRequesterId(storedRequesterId || '');
      setUserRole(storedRoleUser === 'fixer' ? 'fixer' : 'requester');
      setIsLoading(false);
    } else {
      router.push('/');
    }
  }, [router]);
  
  // ...
}
```

<Warning>
You must have a valid fixer_id in session storage to access the calendar.
</Warning>

## Role Switching

Fixers can switch between fixer and requester views:

```typescript
// Location: src/app/[locale]/calendar/page.tsx:57-59
const switchRole = () => {
  setUserRole((prevRole) => (prevRole === 'requester' ? 'fixer' : 'requester'));
};
```

**Switch Button:**

## Appointment Data Management

The calendar uses custom hooks to manage appointment data:

### Six Months Appointments Hook

```typescript
// Location: src/app/[locale]/calendar/page.tsx:73-82
const {
  isHourBookedFixer,
  isHourBooked,
  isEnabled,
  isCanceled,
  refetch: refetchSixMonths,
  refetchHour,
  loading,
} = useSixMonthsAppointments(fixer_id, today);
```

This hook provides:
- `isHourBookedFixer`: Check if fixer has bookings for specific hours
- `isHourBooked`: Check if a customer has booked a time slot
- `isEnabled`: Check if fixer is available at a specific time
- `isCanceled`: Check if an appointment is canceled
- `refetch`: Reload appointment data
- `loading`: Loading state

### Daily Appointments Hook

```typescript
// Location: src/app/[locale]/calendar/page.tsx:84
const { getAppointmentsForDay, refetch: refetchConts } = useDailyConts({ date: today, fixer_id });
```

Provides:
- `getAppointmentsForDay`: Get all appointments for a specific day
- `refetch`: Reload daily appointments

### Refetch All Data

```typescript
// Location: src/app/[locale]/calendar/page.tsx:86-89
const refetchAll = useCallback(() => {
  refetchSixMonths();
  refetchConts();
}, [refetchSixMonths, refetchConts]);
```

## Context Providers

The calendar uses multiple context providers for state management:

### User Role Provider

```typescript
// Location: src/app/[locale]/calendar/page.tsx:123
<UserRoleProvider role={userRole} fixer_id={fixer_id} requester_id={requester_id}>
```

Provides access to:
- Current user role (fixer or requester)
- Fixer ID
- Requester ID

### Appointments Provider

```typescript
// Location: src/app/[locale]/calendar/page.tsx:124-133
<AppointmentsProvider
  isHourBookedFixer={providerValue.isHourBookedFixer}
  isHourBooked={providerValue.isHourBooked}
  isEnabled={providerValue.isEnabled}
  isCanceled={providerValue.isCanceled}
  getAppointmentsForDay={providerValue.getAppointmentsForDay}
  refetchAll={providerValue.refetchAll}
  refetchHour={providerValue.refetchHour}
```

### Appointments Status Provider


## Date Selection

```typescript
// Location: src/app/[locale]/calendar/page.tsx:53-55
const handleDataChange = (newDate: Date) => {
  setSelectedDate(newDate);
};
```

When a user selects a different date:
1. The `selectedDate` state is updated
2. All date-dependent components re-render
3. Appointment data is fetched for the new date
