## Overview

Create a new appointment to schedule service with a fixer.

## Endpoint

```
POST /api/appointments
```

## Request Body

<ParamField body="id_fixer" type="string" required>
  Unique identifier of the fixer
</ParamField>

<ParamField body="id_requester" type="string" required>
  Unique identifier of the requester
</ParamField>

<ParamField body="selected_date" type="string" required>
  Selected date for the appointment (ISO 8601 format)
</ParamField>

<ParamField body="current_requester_name" type="string" required>
  Name of the person requesting the service
</ParamField>

<ParamField body="appointment_type" type="string" required>
  Type of service (e.g., "plomeria", "electricidad")
</ParamField>

<ParamField body="appointment_description" type="string" required>
  Detailed description of the work needed
</ParamField>

<ParamField body="current_requester_phone" type="string" required>
  Contact phone number
</ParamField>

<ParamField body="starting_time" type="string" required>
  Appointment start time (ISO 8601 format)
</ParamField>

<ParamField body="finishing_time" type="string" required>
  Appointment end time (ISO 8601 format)
</ParamField>

<ParamField body="display_name_location" type="string" required>
  Human-readable location name/address
</ParamField>

<ParamField body="lat" type="string">
  Latitude coordinate of the location
</ParamField>

<ParamField body="lon" type="string">
  Longitude coordinate of the location
</ParamField>

<ParamField body="link_id" type="string">
  Optional link to related resources (job offers, etc.)
</ParamField>

## Validation Rules

<AccordionGroup>
  <Accordion title="Date and Time">
    - `starting_time` must be in the future
    - `finishing_time` must be after `starting_time`
    - `selected_date` should match the date portion of `starting_time`
    - Time slots should be reasonable (typically 1-8 hours)
  </Accordion>
  
  <Accordion title="Contact Information">
    - `current_requester_phone` must be a valid phone number format
    - `current_requester_name` is required and must not be empty
  </Accordion>
  
  <Accordion title="Location">
    - `display_name_location` is required
    - `lat` and `lon` are optional but recommended for mapping features
    - If provided, coordinates must be valid (lat: -90 to 90, lon: -180 to 180)
  </Accordion>
  
  <Accordion title="Service Details">
    - `appointment_type` must match available service categories
    - `appointment_description` should be detailed enough for the fixer to understand the work
  </Accordion>
</AccordionGroup>

## Business Logic

### Availability Check

Before creating an appointment, you should check if the fixer is available:

```typescript
// Get existing appointments for the fixer
const existingAppointments = await getAppointmentsByFixerId(fixerId);

// Check for conflicts
const hasConflict = existingAppointments.some(apt => {
  const aptStart = new Date(apt.starting_time.$date);
  const aptEnd = new Date(apt.finishing_time.$date);
  const newStart = new Date(starting_time);
  const newEnd = new Date(finishing_time);
  
  return (
    (newStart >= aptStart && newStart < aptEnd) ||
    (newEnd > aptStart && newEnd <= aptEnd) ||
    (newStart <= aptStart && newEnd >= aptEnd)
  );
});

if (hasConflict) {
  throw new Error('Fixer is not available at the requested time');
}
```

### Notification Flow

After creating an appointment:

1. Send confirmation to the requester
2. Notify the fixer of the new appointment
3. Add to both users' calendars
4. Send reminder notifications before the appointment
